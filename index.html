1. Why Julia?

  1. Julia is easy to code and fast to run!

  2. Fast like C/C++, expressive and dynamic like Python.

  3. "Batteries included": REPL, package manager, multidimensional arrays, linear algebra, plotting.

  4. Free and open source.

  5. Built with scientific computing in mind.

To test the speed, let's try estimating $\pi$ using the formula:

\[ \pi = 4\left(1 - \frac{1}{3} + \frac{1}{5} - \frac{1}{7} + \frac{1}{9} - \frac{1}{11} + \dots\right) \]

julia> function estimate_pi(n)
         s = 1.0
         for i in 1:n
           if i % 2 == 1
             si = -1 / (2i + 1)
           else
             si = 1 / (2i + 1)
           end
           s += si
         end
         return 4s
       end
estimate_pi (generic function with 1 method)

julia> my_pi = estimate_pi(100_000_000)
3.141592663589326

julia> my_pi - pi
9.999532757376528e-9

Here is the same function in C++:

double estimate_pi(int n) {
  double s = 1.0;
  double si;
  for(int i = 1; i < n; i++) {
    if(i % 2 == 1) {
      si = -1.0 / (2.0 * i + 1.0);
    }
    else {
      si = 1.0 / (2.0 * i + 1.0);
    }
    s += si;
  }
  return 4 * s;
}

which takes a similar amount of time.

Variables

julia> x = 2
2

julia> y = 3
3

julia> z = x + y
5

By default, Julia will infer the types of the variables and generate optimal code for the given types (a process called "type inference"). Julia types can be specified (but only inside of functions or other local scopes):

julia> let
         x::Int = 2
         y::Int = 3
         z::Int = x + y
       end
5

This can sometimes have implications for performance if it helps Julia reason about the types better. However, it is usually not necessary since the Julia compiler can infer the types automatically, like the C++ auto keyword:

int main() {
  auto x = 2;
  auto y = 3;
  auto z = x + y;
  return 0;
}

Unicode

In Julia, you can use unicode characters for variables by typing backslash (\) followed by the name of the unicode character (often matching the names in $\LaTeX$) and then followed by pressing the TAB key:

julia> pi
π = 3.1415926535897...

julia> π # \pi TAB
π = 3.1415926535897...

julia> typeof(π)
Irrational{:π}

julia> θ = π / 2 # \theta TAB
1.5707963267948966

You can also use the following shorthand notation for defining functions:

julia> plus1(x) = x + 1

julia> function plus1(x::String)
         return x * " + 1"

This is what is referred to as just-in-time (JIT) compilation, where Julia compiles a specialized function for each type of input.

Generic functions in Julia are similar to C++ templated functions:

template<typename T>
auto plus1(T x) {
  return x + 1;
};

auto plus1(std::string x) {
  return x + " + 1";
};

However, it is easier in Julia to dynamically define overloads of functions with mixtures of different (potentially user-defined) types, as we will see later.

Arrays

Julia has great built-in support for multidimensional arrays.

Julia Arrays can accept any data type, and the storage will be specialized based on the inputs:

julia> numbers = [10, 20, 30, 40]

 30

julia> typeof(numbers)

julia> strings = ["X", "Y", "Z", "W"]
4-element Vector{String}:

julia> typeof(strings)
Vector{String} (alias for Array{String, 1})

   "Y"

   "W"

julia> typeof(any)

Installing packages

Julia has a great built-in package manage, which is accessed through the standard Julia Pkg module that is shipped with Julia.

For example, say we want to use Julia's native symbolic algebra system, Symbolics.jl. We just need to do the following:

julia> using Pkg

julia> Pkg.add("Symbolics")

julia> using Symbolics

julia> @variables x y
2-element Vector{Symbolics.Num}:
 x
 y

julia> z = expand((x + y)^2)
x^2 + y^2 + 2x*y

julia> substitute(z, Dict(y => 3))
9 + x^2 + 6x

Mitigating compilation time

JIT compilation can lead to long initial load times when first loading a package and calling functions from a package. This is particularly noticable for the standard plotting package Plots.jl, as well as automatic differentiation packages like Zygote.jl and calling some functions from our tensor network library, ITensors.jl. This is just a consequence that Julia is a compiled language, so Julia has to compile code just like you would have to compile code in C++, though it happens automatically in the background. However, this can be jarring for users coming from interpreted languages like Python, which does not compile code as aggressively as C++ or Julia at the expense of slower runtimes.

One way to mitigate this issue is to run your code from the REPL and keep the REPL open. You will encounter compilation times the first time you run your code, but if you keep the REPL open those compiled functions will be cached for later use.

Additionally, you can use PackageCompiler.jl (https://julialang.github.io/PackageCompiler.jl/stable/examples/plots.html) which allows for ahead-of-time (AOT) compilation of a set of functions from a Julia package.

Compilation time is a big focus of the core Julia developers and Julia library developers, and tools for performing more automated ahead-of-time compilation will improve as Julia matures!

Custom types

As an example of a custom type, we will define a Polar type, which is a representation of a complex number $z = x + iy$ in the polar representation $z = re^{i\theta}$.

See Wikipedia ( https://en.wikipedia.org/wiki/Complexnumber#Polarcomplex_plane, https://en.wikipedia.org/wiki/Polarcoordinatesystem#Complex_numbers) for more details.

julia> struct Polar <: Number
         r::Float64
         θ::Float64
       end

julia> z1 = Polar(1.2, π / 2)
Main.##WeaveSandBox#343.Polar(1.2, 1.5707963267948966)

julia> Base.show(io::IO, z::Polar) = print(io, z.r, " * exp(", z.θ, "im)")

julia> z1
1.2 * exp(1.5707963267948966im)

Now we will define some operations for our type. For example we can start with multiplication of two polar numbers:

julia> z2 = Polar(3.3, π / 3)
3.3 * exp(1.0471975511965976im)

julia> z1 * z2
Error: * not defined for Main.##WeaveSandBox#343.Polar

julia> import Base: * # Overload the behavior of * for our new type

julia> *(z1::Polar, z2::Polar) = Polar(z1.r * z2.r, z1.θ + z2.θ)
* (generic function with 997 methods)

julia> z1 * z2
3.9599999999999995 * exp(2.617993877991494im)

This allows us to take integer powers of polar numbers:

julia> z1 ^ 3
1.728 * exp(4.71238898038469im)

We can also support multiplying by real numbers:

julia> 2.3 * z1
Error: promotion of types Float64 and Main.##WeaveSandBox#343.Polar failed to change any arguments

julia> *(z1::Real, z2::Polar) = Polar(z1 * z2.r, z2.θ)
* (generic function with 998 methods)

julia> 2.3 * z1
2.76 * exp(1.5707963267948966im)

julia> *(z1::Polar, z2::Real) = Polar(z1.r * z2, z1.θ)
* (generic function with 999 methods)

julia> z1 * 2.3
2.76 * exp(1.5707963267948966im)

This kind of function overloading is referred to as "multiple dispatch": we are defining unique behavior for an existing function for a mixture of different types, which Julia can call dynamically at runtime.

You can see Stefan Karpinski's JuliaCon talk (youtube.com/watch?v=kc9HwsxE1OY) for a deeper dive into multiple dispatch and how it differs from generic template programming in C++ and the object-oriented single dispatch system in Python.

We can even put our new type into other Julia data structures, like a Matrix:

julia> A = [z1 z2; z2 z1]
2×2 Matrix{Main.##WeaveSandBox#343.Polar}:
 1.2 * exp(1.5708im)  3.3 * exp(1.0472im)
 3.3 * exp(1.0472im)  1.2 * exp(1.5708im)

julia> 3 * A
2×2 Matrix{Main.##WeaveSandBox#343.Polar}:
 3.6 * exp(1.5708im)  9.9 * exp(1.0472im)
 9.9 * exp(1.0472im)  3.6 * exp(1.5708im)

Custom Types Exercise

As an exercise, define other operations like division of two Polar numbers (z1 / z2), $n$-th root of Polar numbers z^(1 / n), the addition of two Polar numbers z1 + z2, etc.

Custom Types Exercise Solution

Division of two Polar numbers:

julia> z1 / z2
Error: / not defined for Main.##WeaveSandBox#343.Polar

julia> import Base: /

julia> /(z1::Polar, z2::Polar) = Polar(z1.r / z2.r, z1.θ - z2.θ)
/ (generic function with 319 methods)

julia> z1 / z2
0.36363636363636365 * exp(0.5235987755982989im)

Raise a Polar number to a fractional power:

julia> z1^(1 / 3)
Error: promotion of types Main.##WeaveSandBox#343.Polar and Float64 failed to change any arguments

julia> import Base: ^

julia> ^(z::Polar, n::Float64) = Polar(z.r^n, n * z.θ)
^ (generic function with 241 methods)

julia> z1^(1 / 3)
1.0626585691826111 * exp(0.5235987755982988im)

Add two Polar numbers with +:

julia> z1 + z2
Error: + not defined for Main.##WeaveSandBox#343.Polar

julia> import Base: +

julia> function +(z1::Polar, z2::Polar)
         real = z1.r * cos(z1.θ) + z2.r * cos(z2.θ)
         imag = z1.r * sin(z1.θ) + z2.r * sin(z2.θ)
         z = real + imag * im
         return Polar(abs(z), angle(z))
       end
+ (generic function with 594 methods)

julia> z1 + z2
4.380516088085142 * exp(1.1845996490735033im)

Defining * and + is all that is needed for Julia's generic matrix multiplication to work for matrices of Polar numbers:

julia> A = [z1 z2; z2 z1]
2×2 Matrix{Main.##WeaveSandBox#343.Polar}:
 1.2 * exp(1.5708im)  3.3 * exp(1.0472im)
 3.3 * exp(1.0472im)  1.2 * exp(1.5708im)

julia> A * A
2×2 Matrix{Main.##WeaveSandBox#343.Polar}:
 11.6768 * exp(2.2014im)    7.92 * exp(2.61799im)
   7.92 * exp(2.61799im)  11.6768 * exp(2.2014im)

julia> A ^ 3
2×2 Matrix{Main.##WeaveSandBox#343.Polar}:
 40.0959 * exp(-2.58066im)   44.7998 * exp(-2.8624im)
  44.7998 * exp(-2.8624im)  40.0959 * exp(-2.58066im)

For efficiency, it may be better to allow +(::Polar, ::Polar) return a ComplexF64, since converting back to the Polar form adds some extra cost. In general, the Polar form will be faster for multiplication while the traditional complex representation will be faster for addition.

6. Advanced topics

Calling Python from Julia

Julia has great functionality for calling out to other languages, such as C, Fortran, Python, and R.

For calling out to Python, you can use PythonCall.jl (which is the next generation version of PyCall.jl).

For example:

julia> using Pkg

julia> Pkg.add("PythonCall")

julia> using PythonCall

julia> Pkg.add("CondaPkg")

julia> using CondaPkg

julia> CondaPkg.add("numpy")

julia> np = pyimport("numpy")
Python module: <module 'numpy' from '/home/mfishman/.julia/environments/v1.7/.CondaPkg/env/lib/python3.10/site-packages/numpy/__init__.py'>

 3  4
